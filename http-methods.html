HTTP Methods

Overview of common verbs and their properties (safe, idempotent, cacheable).

GET

Retrieve a representation; no body semantics; safe, idempotent, cacheable.

POST

Create or process a subordinate resource; not idempotent; conditionally cacheable.

PUT

Replace a resource at a known URI; idempotent; not cacheable by default.

PATCH

Apply a partial modification; idempotency depends on patch semantics.

DELETE

Remove a resource; idempotent by intent; not cacheable.

HEAD

Like GET but without response body; safe, idempotent, cacheable.

OPTIONS

Describe communication options; safe, idempotent.

TRACE

Echo request; used for diagnostics; safe/idempotent, often disabled.

CONNECT

Establish a tunnel to the server; used with proxies (e.g., HTTPS).
