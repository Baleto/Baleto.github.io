Security & HTTPS

HTTPS is simply HTTP over TLS (Transport Layer Security). The request and response semantics are the same, but the transport is encrypted and integrity-protected.

Modern browsers increasingly assume HTTPS by default: they highlight insecure origins, block mixed content, and integrate certificate checks into the user‚Äôs security experience.

  • Goal: Protect confidentiality, integrity, and authenticity of web traffic.
  • Key building blocks: TLS handshake, certificates, CAs, HSTS, modern ciphers.

How HTTPS Works (High Level)

HTTPS relies on TLS, a protocol that runs beneath HTTP. Before the browser sends an HTTP request, it performs a TLS handshake with the server:

  1. The client says ‚Äúhello‚Äù and offers supported cipher suites and protocol versions.
  2. The server chooses a cipher suite and sends its certificate.
  3. The client validates the certificate chain against trusted Certificate Authorities (CAs), plus hostname and validity checks.
  4. Client and server derive shared keys and confirm that both sides agree on them.
  5. Once the handshake succeeds, HTTP messages are sent inside the encrypted tunnel.

The result is a secure channel: observers can‚Äôt read the content, and tampering is detectable.

Key Security & HTTPS Concepts

Important terms you will see in HTTP security discussions.
Concept Summary
TLS (Transport Layer Security) Cryptographic protocol that encrypts and authenticates data in transit. HTTPS is HTTP carried over a TLS session.
Certificate A signed document binding a public key to a domain name. Issued by a Certificate Authority (CA) and validated by the browser.
Certificate Authority (CA) A trusted third party that issues certificates. Browsers ship with a root store of trusted CAs.
HSTS (HTTP Strict Transport Security) A response header (Strict-Transport-Security) that tells browsers to always use HTTPS for a given origin and optionally subdomains.
Mixed Content When an HTTPS page loads HTTP resources (images, scripts, etc.). Modern browsers block or warn about this because it weakens the security of the page.
Perfect Forward Secrecy (PFS) A property of modern cipher suites: even if a server key is compromised later, past sessions remain confidential.

In practice: always use HTTPS for production sites, redirect HTTP to HTTPS, enable HSTS when you are confident in your configuration, and keep certificates renewed and automated.
