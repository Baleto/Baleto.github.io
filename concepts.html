Core Concepts

  1. Application Layer Role. HTTP operates at the application layer, enabling web clients and servers to exchange data using a standardized request/response model.
  2. Statelessness. Each request is independent. State is carried by tokens or cookies rather than the protocol itself.
  3. Request & Response. A request line + headers + optional body; the server replies with a status line + headers + optional body.
  4. Headers. Metadata that governs behavior (e.g., Content-Type, Authorization, Cache-Control).
  5. HTTP vs HTTPS. HTTPS is HTTP over TLS, providing confidentiality, integrity, and server authenticity.
  6. Versions. HTTP/1.1 (persistent connections), HTTP/2 (binary framing/multiplexing), HTTP/3 (QUIC/UDP transport).
  7. Caching (Brief). Validators and cache directives reduce latency and bandwidth when used correctly.
